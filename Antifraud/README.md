# Детекция сгенерированного текста

Проект для обнаружения сгенерированного (машинного) текста на основе Jupyter Notebook и текстовых датасетов.

## Описание проекта

Цель проекта — **определять, был ли текст написан человеком или сгенерирован моделью**.  
Для этого используется следующий общий подход:

- загрузка размеченного датасета с примерами человеческих и машинных текстов;
- предварительная обработка и анализ данных;
- извлечение признаков с помощью современных языковых моделей;
- обучение классического ML-классификатора;
- оценка качества модели с помощью стандартных метрик (accuracy, F1, ROC-AUC и др.) и визуализаций.

Все шаги реализованы в одном Jupyter Notebook, что делает эксперимент легко воспроизводимым и удобным для модификации.

## Структура проекта

В репозитории находятся:

- `Generated_Text_Detection.ipynb` — **основной ноутбук проекта**. Содержит:
  - установку и импорт необходимых библиотек;
  - загрузку и подготовку данных;
  - исследовательский анализ данных (EDA) и визуализацию;
  - построение и обучение модели детекции сгенерированного текста;
  - вычисление и вывод метрик качества, построение графиков.
- `*.txt` — **два текстовых файла с датасетами**. Как правило, это файлы с примерами текстов и их метками (человеческий/сгенерированный текст).  
  Точный формат и разметка определяются содержимым файлов; при необходимости загляните внутрь, чтобы понять структуру (например, разделитель колонок, наличие заголовков и т.п.).

## Требования

### Версия Python

- **Python**: рекомендуется версия **3.10+**.

### Основные зависимости

Для запуска ноутбука потребуются:

- **Jupyter**:
  - `jupyter` или `jupyterlab`
- **Библиотеки для работы с ML/NLP и данными**:
  - `transformers`
  - `datasets`
  - `torch` (с поддержкой CPU или GPU, в зависимости от доступного оборудования)
  - `catboost`
  - `scikit-learn`
  - `pandas`
  - `numpy`
  - `matplotlib`
  - `seaborn`
  - `tqdm`

Установка этих библиотек может выполняться как через отдельный `requirements.txt`, так и напрямую командами `pip` (см. раздел ниже).

## Установка и запуск

Ниже описан базовый сценарий запуска на локальной машине (Windows).

### 1. Клонирование репозитория

Выполните в терминале (PowerShell или CMD):

```bash
git clone <URL_репозитория>
cd Generated_Text_Detection
```

Замените `<URL_репозитория>` на фактический URL вашего репозитория (например, с GitHub или другого хостинга).

### 2. Создание и активация виртуального окружения (рекомендуется)

```bash
python -m venv .venv
.venv\Scripts\activate
```

После активации виртуального окружения в начале строки терминала обычно появляется префикс `(.venv)` или похожий.

### 3. Установка зависимостей

Обновите `pip` и установите необходимые библиотеки:

```bash
pip install --upgrade pip
pip install jupyter
pip install transformers datasets torch catboost scikit-learn pandas numpy matplotlib seaborn tqdm
```

При необходимости вы можете дополнительно настроить установку `torch` под вашу версию CUDA или использовать CPU-версию (см. официальную документацию PyTorch).


### Работа с ноутбуком

1. Откройте `Generated_Text_Detection.ipynb` в интерфейсе Jupyter.
2. Последовательно запускайте ячейки сверху вниз:
   - сначала ячейки с установкой зависимостей (при необходимости) и импортом библиотек;
   - затем ячейки, загружающие датасеты из текстовых файлов;
   - блоки с предварительной обработкой и анализом данных;
   - ячейки с обучением модели и расчётом метрик;
   - ячейки с визуализацией результатов (графики, матрица ошибок и т.п.).
3. По завершении обучения вы получите:
   - метрики качества (accuracy, F1, precision, recall и др.);
   - отчёт по классификации;
   - графические визуализации, помогающие понять поведение модели.

### Повторные эксперименты

- При желании вы можете:
  - изменить параметры выборки данных;
  - заменить или добавить другие датасеты;
  - настроить гиперпараметры модели;
  - использовать другие модели или признаки.

Все такие изменения вносятся напрямую в ячейки ноутбука.
