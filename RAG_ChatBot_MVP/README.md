# RAG-чат-бот для базы знаний (прототип)

Чат-бот помогает новым сотрудникам изучать техническую документацию: отвечает на вопросы на русском с опорой на PDF из внутренней базы знаний.

---

## 1. Архитектура

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│   PDF (data/    │     │     Ingest       │     │   ChromaDB      │
│   pdf/*.pdf)    │────▶│ Load → Chunk     │────▶│ (локально)     │
└─────────────────┘     │ + метаданные     │     │ векторы +       │
                        └──────────────────┘     │ metadata        │
                                                 └────────▲────────┘
                                                          │
┌─────────────────┐     ┌──────────────────────────────┐  │
│   Пользователь  │     │  Классификация вопроса       │  │
│   "Вопрос"      │────▶│  (LLM: общий / по ЕСКД)      │──│
└─────────────────┘     └──────────────┬───────────────┘  
                                       │                  
                      ┌────────────────┴─────────────┐
                      │  Вопрос по ЕСКД → RAG        │
                      │  Embed query → Retrieve →    │───────▶ Ответ + источники
                      │  LLM (qwen2.5:7b-instruct)   │
                      └──────────────────────────────┘

                      ┌──────────────────────────────┐
                      │  Общий вопрос → LLM          │
                      │  Прямой ответ без RAG        │───────▶ Ответ (без ссылок)
                      │  (qwen2.5:7b-instruct)       │
                      └──────────────────────────────┘
```

**Этапы:**

| Этап | Описание |
|------|----------|
| **Ingestion** | Загрузка PDF (pypdf), разбиение на чанки с учётом структуры, эмбеддинг через Ollama (модель `qwen3-embedding:0.6b`, 1024‑мерные векторы), сохранение в ChromaDB |
| **Классификация вопроса** | Определение типа вопроса LLM (`qwen2.5:7b-instruct`): **«Вопрос по ЕСКД»** или **«Общий вопрос»**; от результата зависит, будет ли запускаться RAG-пайплайн |
| **Retrieval (для вопросов по ЕСКД)** | Эмбеддинг запроса, поиск по косинусной близости в ChromaDB, возврат top‑k чанков с метаданными (`source`, `page`, `section`) |
| **Generation (RAG)** | Формирование контекста из чанков и метаданных, промпт, генерация детализированного ответа через Ollama (`qwen2.5:7b-instruct`) с указанием источников |
| **Generation (без RAG)** | Для общих вопросов LLM (`qwen2.5:7b-instruct`) даёт прямой ответ без обращения к векторной базе, с указанием, что вопрос вне тематики ЕСКД/ГОСТ |

## 3. Модели

| Назначение | Модель | Обоснование |
|------------|--------|-------------|
| **Эмбеддинги** | Ollama + `qwen3-embedding:0.6b` | Бесплатные локальные эмбеддинги через Ollama, поддержка русского, 1024‑мерные векторы, интеграция через `OllamaEmbeddings` и ChromaDB |
| **LLM** | Ollama + `qwen2.5:7b-instruct` | Бесплатная локальная LLM через Ollama, хорошее качество на русском, используется и для классификации вопросов, и для генерации ответов |

**Требования:**  
- Ollama установлен и запущен.
- Загружены модели: `qwen2.5:7b-instruct` и `qwen3-embedding:0.6b` (см. раздел 6.2).

---

## 4. Chunking

- **Стратегия:** RecursiveCharacterTextSplitter с разделителями  
  `["\n\n## ", "\n\n### ", "\n\n", "\n", " "]` — приоритет разделам, затем абзацам.
- **chunk_size=800, overlap=120** — баланс между контекстом и точностью retrieval; при необходимости меняется в `config.py`.
- **Метаданные чанка:** `source`, `page`, `section` (section — эвристика по первой строке: короткая, без `.!?`).

---

## 5. ChromaDB: схема коллекции

- **Имя:** `kb_docs`
- **Персистентность:** `chroma_data/` (persist_directory)
- **Метаданные:** `source`, `page`, `section` — для вывода в ответ и (при расширении) фильтрации.

ChromaDB работает полностью локально, без отдельного сервера.

## 6. Запуск

### 6.1. Окружение и зависимости

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

### 6.2. Ollama (LLM)

Чат-бот обращается к локальному Ollama по умолчанию (http://localhost:11434). Нужны: установленный Ollama и загруженные модели.

1. **Установка:** скачайте установщик с [ollama.com](https://ollama.com/download) и установите Ollama. Запустите приложение «Ollama» из меню Пуск — в трее появится иконка, сервер будет работать.

2. **Модели:** один раз нужно скачать модели, используемые проектом:
   - Если в терминале команда `ollama` доступна (Ollama добавлен в PATH):  
     `ollama pull qwen2.5:7b-instruct`  
     `ollama pull qwen3-embedding:0.6b`
   - Если команда не найдена (типично для Windows): откройте в браузере http://localhost:11434, в веб-интерфейсе выберите модели **Qwen2.5 7B Instruct** и **Qwen3 Embedding 0.6B** и нажмите Download, либо добавьте путь к Ollama в PATH (папка установки, например `C:\Users\<user>\AppData\Local\Programs\Ollama`) и выполните в терминале команды выше.

3. **Проверка:** пока приложение Ollama запущено и модели загружены, чат-бот сможет вызывать LLM и эмбеддинги. Запуск интерактивного чата через `ollama run ...` не обязателен — для RAG достаточно работающего сервера Ollama и указанных моделей.

### 6.3. PDF в data/pdf

Положить нужные PDF в `data/pdf/`.

### 6.4. Индексация

```bash
python main.py ingest
```

### 6.5. Вопрос к базе (CLI)

```bash
python main.py ask "Как оформить отпуск?"
```

### 6.6. Telegram-бот

1. Создайте бота в Telegram через [@BotFather](https://t.me/BotFather), получите токен.
2. Скопируйте `.env.example` в `.env` и укажите:
   ```
   TELEGRAM_BOT_TOKEN=ваш_токен
   RUN_MODE=telegram
   ```
3. Запуск бота:
   ```bash
   python main.py telegram
   ```
   либо при `RUN_MODE=telegram` в `.env`:
   ```bash
   python main.py run
   ```
   Бот будет отвечать на текстовые сообщения, используя тот же RAG-пайплайн, что и CLI.

## 7. Пример запроса и ответа

**Запрос:**

```
python main.py ask "Как настроить VPN для удалённого доступа?"
```

**Пример ответа (при наличии в PDF соответствующих разделов):**

> Для настройки VPN для удалённого доступа нужно установить корпоративный клиент из портала ИТ, выбрать тип подключения «Удалённый офис» и ввести логин и пароль от домена. После первого входа система запросит двухфакторную аутентификацию.
>
> **Источники:** Инструкция_ИТ.pdf (стр. 4), Онбординг_новых_сотрудников.pdf (стр. 2)

## 8. Ограничения прототипа

- Нет авторизации, масштабирования, мониторинга, отказоустойчивости.
- Используются только бесплатные, локальные компоненты; облачные сервисы и платные API не применяются.
- Код рассчитан на Python 3.10+.

---

## 9. Расширяемость

- **Фильтрация по метаданным:** в `as_retriever` можно задать `search_kwargs={"filter": {"source": "..."}}` по ChromaDB.
- **Другие форматы:** в `ingest` добавить загрузчики (DOCX, MD) и общий `chunk_documents`.
- **Смена LLM/эмбеддингов:** в `config` и в `store`/`rag` поменять модель и конструирование клиентов.
