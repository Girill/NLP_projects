## Название проекта

**User Preference Classification — Best Model Answer**

## Описание проекта

Проект решает задачу **классификации «лучшего ответа» модели** на пользовательский запрос.  
На вход подаются:

- текст запроса пользователя;
- несколько ответов (обычно два) от разных языковых моделей;
- метка, какой из ответов был признан лучшим (или что ответы равны).

В ноутбуке реализован пайплайн:

- подготовка и анализ датасета с пользовательскими предпочтениями;
- формирование текстовых примеров вида «запрос + ответы»;
- дообучение предобученной трансформер‑модели (например, DistilBERT) на задачу классификации лучшего ответа;
- оценка качества и применение модели для инференса;
- (опционально) дообучение через LoRA‑адаптеры с использованием библиотеки PEFT.

Цель проекта — дать практический пример того, как можно **обучить модель оценивать и ранжировать ответы** по предпочтениям пользователей, используя публичные инструменты Hugging Face и PyTorch.

## Структура проекта

В репозитории содержатся следующие файлы:

### Jupyter Notebook

- `Best_Model_Answer_Classificator.ipynb`  
  Основной ноутбук проекта. В нём реализованы:

  - загрузка и предварительный анализ данных;
  - построение признаков (объединение запроса и ответов в одну строку для токенизации);
  - класс `ModelAnswerClassifier` на базе `transformers` (загрузка модели, токенизация, обучение, валидация, инференс, сохранение/загрузка модели);
  - обучение модели на обучающем датасете и оценка на валидационном;
  - пример применения модели к тестовому датасету и/или новым примерам;
  - (опционально) использование LoRA через PEFT для параметро‑эффективного дообучения.

### Датасеты

- `train_user_preference.csv`  
  Текстовый файл (CSV) с обучающим набором данных. Содержит:

  - идентификатор примера;
  - идентификаторы/названия моделей, сгенерировавших ответы;
  - текст запроса (`prompt`);
  - несколько столбцов с ответами (например, `response_a`, `response_b` или аналогичные);
  - бинарные/много-классовые метки предпочтений (например, какой ответ победил, либо что есть ничья).

- `test_user_preference.csv`  
  Текстовый файл (CSV) с тестовым набором данных в похожем формате, но без меток предпочтений или с сокращённым набором столбцов. Используется для применения обученного классификатора к новым примерам и формирования предсказаний.

> Фактические имена и количество столбцов в CSV лучше уточнить, открыв файлы в ноутбуке или любым табличным редактором — README описывает структуру обобщённо.

## Использование

### 1. Подготовка датасетов

Убедитесь, что:

- файлы `train_user_preference.csv` и `test_user_preference.csv` находятся в той же директории, что и ноутбук;
- пути к файлам в ячейке загрузки данных соответствуют их реальному расположению.  
  Пример (как это сделано в ноутбуке):

```python
train_df = pd.read_csv('train_user_preference.csv')
test_df = pd.read_csv('test_user_preference.csv')
```

Если файлы лежат в другой папке, укажите относительный или абсолютный путь.

### 2. Запуск ячеек ноутбука

Рекомендуемый порядок работы:

1. **Импорт библиотек**  
   Запустите верхние ячейки, которые импортируют библиотеки и устанавливают базовые параметры (включая случайные seed’ы, конфигурацию модели и т.д.).

2. **Загрузка и анализ данных**  
   Выполните ячейки, которые:
   - читают `train_user_preference.csv` и `test_user_preference.csv`;
   - выводят `.info()`, `.head()` и простую статистику по датасету;
   - строят базовые графики (распределения длин текстов, классов и т.п.), если предусмотрено.

3. **Создание и инициализация классификатора**  
   Найдите и запустите ячейку, где создаётся объект `ModelAnswerClassifier`, например:

   ```python
   classifier = ModelAnswerClassifier(
       model_name="distilbert-base-uncased",
       num_labels=3,
       num_answers=2,
       batch_size=32,
       learning_rate=2e-4,
       num_epochs=10,
       max_length=512,
       warmup_steps=100,
       freeze_params=True,
       # опционально:
       # use_lora=True,
       # lora_r=8,
       # lora_alpha=16,
       # lora_dropout=0.1,
   )

   classifier.prepare_model()
   ```

   Параметры можно адаптировать под вашу задачу и ресурсы.

4. **Подготовка датасетов для обучения**  
   Выполните ячейки, в которых:

   - данные разбиваются на обучающую/валидационную выборку (`train_test_split`);
   - создаются экземпляры датасета через метод `create_dataset(...)` классификатора;
   - формируются `train_dataset` и `val_dataset`.

5. **Обучение модели**

   Запустите ячейку/ячейки, в которых вызывается:

   ```python
   history = classifier.train(train_dataset, val_dataset)
   ```

   В процессе вы увидите:
   - логи по эпохам (loss, accuracy),
   - при наличии — метрики на валидации.

6. **Оценка и инференс**

   После обучения вы можете:

   - оценить модель на отдельном датасете, используя `classifier.evaluate(...)`;
   - сделать предсказания на `test_user_preference.csv` или новых данных:

   ```python
   predictions, probs = classifier.predict(
       prompts=test_prompts,
       answers=test_answers,
       return_probs=True,
   )
   ```

   Формат входных списков (`prompts`, `answers`) в ноутбуке уже подготовлен — достаточно следовать существующим примерам.

7. **Сохранение и загрузка модели**

   В ноутбуке есть методы:

   ```python
   classifier.save_model("path_to_save_dir")
   # ...
   classifier.load_model("path_to_save_dir")
   ```

   - При **обычном обучении** (без LoRA) сохраняется полная модель и токенизатор.
   - При **обучении с LoRA** сохраняются LoRA‑адаптеры и токенизатор; при загрузке автоматически восстанавливается базовая модель и подключаются адаптеры.

### 3. Работа с LoRA (опционально)

Если вы хотите использовать parameter-efficient fine-tuning:

1. Убедитесь, что установлен пакет `peft`:

   ```bash
   pip install peft
   ```

2. При инициализации `ModelAnswerClassifier` передайте `use_lora=True` и нужные гиперпараметры (`lora_r`, `lora_alpha`, `lora_target_modules` и т.д.).

3. Остальной код (подготовка датасета, обучение, инференс, сохранение/загрузка) остаётся прежним.
